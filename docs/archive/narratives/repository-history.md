# 仓库历程

## 范围

这份叙事基于 `2026-04-25` 时 `HEAD` 可达的完整 Git 历史整理而成：总计 `734` 个提交，覆盖时间从 `2026-03-11` 到 `2026-04-24`。

简要来说，SkillHub 不是按缓慢、薄层的方式长出来的。它是在一个高度压缩的时间窗口里，同时完成了设计、实现、工作流搭建、发布准备和快速硬化。第一个月基本确定了产品边界，也搭起了大部分系统表面；第二个月则明显转向文档、运维韧性、兼容性和真实使用场景下的打磨。

主要来源：

- [仓库提交历史来源](../sources/commits/2026-04-25-repository-commit-history.md)

## 阶段一：启动与压缩式成型

`2026-03-11` 到 `2026-03-13`

仓库一开始就呈现出异常密集的创建节奏。`2026-03-11` 落下初始提交，随后紧接着完成项目更名、Maven 多模块后端结构、Spring Boot 入口、认证与 RBAC 基础，以及最早一版前端脚手架。

到了 `2026-03-12`，提交流从“简单骨架”迅速扩展成几乎完整的一张产品相位图。仅在这一天，仓库里就已经落下了：

- 命名空间与技能领域模型
- 包解析与校验
- 对象存储抽象
- 审核与提升服务
- 社交能力
- Device Flow 支持
- ClawHub 兼容层
- 管理员 API
- 运行时镜像发布
- README 和架构文档的大幅更新

这一天的重要性在于，它确立了项目后续一个反复出现的模式：设计文档与实现紧密靠拢，完整功能线往往以成组的方式落地，而不是拆成彼此无关的小碎片。

## 阶段二：产品化、治理与硬化

`2026-03-14` 到 `2026-03-20`

在第一轮搭骨架之后，仓库开始明显转向产品硬化。这个阶段虽然充满了大量 `fix`，但它们并不是随机清理，而是在把系统真正推过一系列具体工作流：

- 预发环境与 PR 工作流
- release compose 与运行时引导脚本打磨
- 落地页、i18n 和导航完善
- 命名空间治理与生命周期管理
- 举报流程与治理中心 UI
- 审核与提升边界修正
- 搜索调优与语义重排基础
- 围绕认证、指标和下载行为的安全加固
- 构建与测试入口统一

这也是项目对外身份逐渐变清晰的时期。README 被反复改写，中文文档加入，集成指引出现，PR 合并开始在历史里形成明显节奏。项目在 `2026-03-19` 达到一个明确节点：`chore(release): v0.1.0`。

## 阶段三：v0.1.0 之后的能力扩展

`2026-03-23` 到 `2026-03-31`

`v0.1.0` 发布后，仓库并没有降速，反而开始横向扩展。接下来一波可见变化，让平台逐步从“核心注册中心可用”走向“围绕注册中心的生态开始成形”：

- 站内通知
- 多扫描器安全扫描能力
- 文件浏览侧栏、预览与审计 UI
- Scanner 的 Redis 和对象存储修复
- 工作流和开发流程改进
- 技能分享
- 更宽松的包类型支持
- 发布部署面继续扩展
- Unicode slug 支持

这一阶段还显示出比最初启动期更广的贡献者参与面。

## 阶段四：文档成熟与真实使用场景稳定化

`2026-04-01` 到 `2026-04-24`

4 月的重心已经明显不再是单纯扩展功能面，而是让产品更好地被运行、被验证、被使用。主要线索包括：

- 双语 VitePress 文档站点
- 更清晰的运行时和 Kubernetes 部署指引
- 更强的 e2e 与 PR 校验流程
- 面向中国开发者的专项说明
- issue 分诊自动化
- 针对搜索、审核、认证、校验和存储的大量反复修正
- 发布预检警告确认流程
- `UPLOADED` 这样的新生命周期语义
- 密码重置和 SMTP 支持
- GitLab OAuth Provider 支持
- S3 惰性建桶与重试处理
- 用于更新既有技能的 Dashboard 体验优化

4 月的提交结构意味着维护者已经在持续碰到真实边界场景，并围绕这些边界把产品不断收紧。搜索可见性、兼容层行为、发布流程和存储语义，都出现了多轮纠正。

## 提交流透露出的项目信号

这里面最显眼的有三个模式。

第一，项目诞生于一次高度压缩的实现冲刺。大多数仓库会在“薄骨架”状态里停留更久，而 SkillHub 几乎是立刻进入全栈功能建设。

第二，提交历史呈现出很强的跨职能特征。UI、认证、文档、治理、兼容性和运维是一起生长的，项目从一开始就不只是一个“后端注册中心”。

第三，历史很快转向协作化。早期提交主要集中在少数贡献者手中，随后逐步转向更明确的 PR 协作节奏，仓库也几乎同步开始沉淀流程知识、文档和自动化。
